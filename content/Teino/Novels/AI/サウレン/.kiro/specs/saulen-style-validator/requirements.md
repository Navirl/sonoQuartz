# サウレンシリーズ文体検証システム 要件定義書

## 概要

サウレンシリーズの小説が既定のガイドラインに準拠しているかを自動的に検証するシステムを構築する。このシステムは新規作成された小説や既存小説の修正版が、キャラクター性、文体、関係性の一貫性を保っているかをチェックする。

## 要件

### 要件1: 文体形式の検証

**ユーザーストーリー:** ライターとして、私の書いた小説が正しい文体形式（会話文と地の文の形式）に従っているかを確認したい。

#### 受入基準

1. WHEN 小説ファイルが検証システムに入力される THEN システムは会話文が `キャラクター名：「セリフ」` 形式になっているかをチェックする
2. WHEN 小説ファイルが検証システムに入力される THEN システムは地の文が `*地の文*` 形式になっているかをチェックする
3. WHEN 不正な形式が検出される THEN システムは具体的な行番号と修正提案を提供する
4. WHEN 全ての形式が正しい THEN システムは「文体形式: 合格」を返す

### 要件2: キャラクター性の一貫性検証

**ユーザーストーリー:** ライターとして、サウレンとyouのキャラクター性が一貫して表現されているかを確認したい。

#### 受入基準

1. WHEN サウレンの台詞が検証される THEN システムは丁寧語の使用をチェックする
2. WHEN サウレンの台詞が検証される THEN システムは職業的報告と感情的反応のバランスをチェックする
3. WHEN youの台詞が検証される THEN システムは冷静で論理的な表現をチェックする
4. WHEN youの描写が検出される THEN システムは物理的描写の禁止ルール違反を警告する
5. WHEN キャラクター性に問題がある THEN システムは具体的な改善提案を提供する

### 要件3: 関係性の表現検証

**ユーザーストーリー:** ライターとして、サウレンとyouの関係性が適切に表現されているかを確認したい。

#### 受入基準

1. WHEN 小説が検証される THEN システムは通信ベースの関係性が維持されているかをチェックする
2. WHEN 小説が検証される THEN システムは二人の信頼関係が表現されているかをチェックする
3. WHEN 直接的な接触や対面の描写が検出される THEN システムは警告を発する
4. WHEN 関係性の表現が適切 THEN システムは「関係性表現: 合格」を返す

### 要件4: 物語構造の検証

**ユーザーストーリー:** ライターとして、物語が基本的な構造パターンに従っているかを確認したい。

#### 受入基準

1. WHEN 小説が検証される THEN システムは転送→探索→発見→危機→解決の基本構造をチェックする
2. WHEN 構造要素が不足している THEN システムは不足要素を指摘する
3. WHEN オープンエンドになっていない THEN システムは改善提案を行う
4. WHEN 構造が適切 THEN システムは「物語構造: 合格」を返す

### 要件5: 専門用語と描写の品質検証

**ユーザーストーリー:** ライターとして、科学的描写や環境描写が適切な品質を保っているかを確認したい。

#### 受入基準

1. WHEN 環境描写が検証される THEN システムは具体性と詩的表現のバランスをチェックする
2. WHEN 科学的用語が使用される THEN システムは適切性をチェックする
3. WHEN 描写が曖昧すぎる THEN システムは具体性向上の提案を行う
4. WHEN 描写品質が適切 THEN システムは「描写品質: 合格」を返す

### 要件6: 総合評価とレポート生成

**ユーザーストーリー:** ライターとして、全体的な評価結果と改善点を分かりやすいレポートで受け取りたい。

#### 受入基準

1. WHEN 全ての検証が完了する THEN システムは総合スコア（0-100点）を算出する
2. WHEN レポートが生成される THEN システムは各項目の詳細結果を含める
3. WHEN 問題が検出される THEN システムは優先度付きの改善提案リストを提供する
4. WHEN 合格基準（80点以上）に達する THEN システムは「サウレンシリーズ準拠: 合格」を表示する

### 要件7: 使いやすいインターフェース

**ユーザーストーリー:** ライターとして、簡単なコマンドで検証を実行し、結果を確認したい。

#### 受入基準

1. WHEN `validate-saulen [ファイル名]` コマンドが実行される THEN システムは指定ファイルの検証を開始する
2. WHEN 検証中 THEN システムは進行状況を表示する
3. WHEN 検証完了 THEN システムは結果を色分けして表示する（合格=緑、警告=黄、エラー=赤）
4. WHEN `--fix` オプションが指定される THEN システムは自動修正可能な項目を修正する

## 成功基準

- 既存のサウレンシリーズ小説（5.md, 6.md, 7.md, 8.md）で90点以上を獲得
- 意図的に作成した不適切な小説サンプルで適切にエラーを検出
- 検証時間が1ファイルあたり5秒以内
- 修正提案の精度が80%以上（人間の判断と一致）

## 制約事項

- 日本語テキストの自然言語処理に対応する必要がある
- ガイドラインの更新に柔軟に対応できる設計とする
- 既存のKiro環境との統合を前提とする
